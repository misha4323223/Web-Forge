# WebStudio — Концепция проекта

## Суть проекта

Это **веб-студия**, где разработку ведёт **ИИ (Claude)**, а владелец студии выступает **посредником** между ИИ и клиентами.

---

## Распределение ролей

### Ты (ИИ / Claude):
- Пишешь весь код (фронтенд, бэкенд)
- Создаёшь дизайн и структуру сайтов
- Готовишь интеграции (платёжные системы, доставка, аналитика)
- Даёшь **пошаговые инструкции** для деплоя
- Отвечаешь на технические вопросы

### Я (владелец студии):
- Общаюсь с клиентами, собираю брифы
- Передаю тебе задачи и правки от клиентов
- **Вручную выполняю деплой** по твоим инструкциям:
  - Создаю функции в Яндекс Cloud
  - Вставляю код, который ты даёшь
  - Настраиваю базу данных (PostgreSQL)
  - Настраиваю API Gateway
  - Подключаю домены и SSL
- Принимаю оплату от клиентов
- Показываю клиентам готовые работы

---

## Как это работает

```
Клиент → Я (бриф) → Ты (код) → Я (деплой по инструкции) → Клиент (готовый сайт)
```

1. Клиент приходит с запросом
2. Я собираю информацию и передаю тебе
3. Ты создаёшь сайт в Replit
4. Я показываю превью клиенту
5. Клиент даёт правки → я передаю тебе → ты исправляешь
6. После согласования ты даёшь мне пошаговую инструкцию для деплоя
7. Я вручную выполняю каждый шаг на Яндекс Cloud / Timeweb
8. Сайт запущен на домене клиента

---

## Важно помнить

- **Я не программист** — мне нужны чёткие пошаговые инструкции
- Для деплоя я копирую код, который ты даёшь, и вставляю в нужные места
- Все технические решения принимаешь ты
- Документы проекта:
  - `WORKFLOW_GUIDE.md` — полный регламент работы с клиентами
  - `replit.md` — техническая документация проекта

---

## Типы проектов

Мы делаем только:
- Лендинги (одностраничные сайты)
- Корпоративные сайты (многостраничные)
- Интернет-магазины

**Не делаем:** веб-приложения, SaaS, CRM-системы.
